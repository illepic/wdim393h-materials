Groups

This class will feature a semester-long group project that is used to turn in each homework. You will be randomly assigned to a group. Homework is turned in per group - that is, each member of a group gets the same grade. A single, representative member of the group will turn in the assignment on the website, usually by submitting a link using the handy form.

Each homework adds sections to a website that your team hosts and maintains. I'll leave it up to you how you split duties (Do you all email files around? GitHub? SSH and Vim?). The homework should make it clear what is required.

For the love of DOM, use all those skills you picked up here to make it pretty. Pretties count on the grade.

A Brief History of the Semantic Web

  • Hyper Text

    “Hypertext” is text that avoids strictly linear organization in favor of hyper-linking to a potentially infinite number of other bits of text.

  • Markup Language

    • Annotate a document, in put in indicators that there's more going on in the background besides just the words shown to the user.

    • "Similarly to how we use quotation marks to denote when someone is saying something "

    • (And parentheses to provide little asides or non-essential information)

HTML Definition

Purpose of HTML is to write text that links to other bits of text, annotated so that the information has organization and context

II Christopher, Chapter 6 Verse 12

<html>
  <body>
  <h1>Lorem Ipsum Dolor</h1>
  <p class="main-content">Placerat, scelerisque, nisi ridiculus ac ut, ac  <a href="http://www.google.com">dignissim</a> phasellus dignissim dignissim augue tristique integer.</p>
  </body>
</html>

Semantic Web

HTML supports the idea of a Semantic Web, where the meaning and context of information is clear.

II Christopher, Chapter 6 Verse 13 HELLO WORLD

<item rdf:about="http://dbpedia.org/resource/Cat">Cat</item>
Example RDF (Rich Data Format) for true data meaning.

Accessibility

As well as Accessible Design, wherein that meaning and context is able to be interpreted by browsers or other devices

II Christopher, Chapter 6 Verse 14

<header role="banner">

<aside id="demo-aside-content" role="complementary">This is just a demo page to see HTML5 markup and WAI-ARIA landmark roles in action in a simple context</aside>

The Birth of the W3C

Rising out of the catastrophe of table-based layouts and browser-specific technology, the W3C is formed by internet-inventor Tim Berners-Lee in 1994.

... made up of member organizations which maintain full-time staff for the purpose of working together in the development of standards for the World Wide Web.

http://en.wikipedia.org/wiki/World_Wide_Web_Consortium

New Standards, New Technology

Cascading Style Sheets

Division of form and content. Information itself could remain distinct, while the experience could be tweaked to suit various devices.

New Standards, New Technology

Javascript

Form, content and function are now fully discreet ... as an afterthought to where we started: HTML and simple data display.

HTML5 IS HERE

HTML5 is a complete package (if not "complete" yet)

Instead of three different technologies growing to fit web needs ad hoc, HTML5 can be defined as the combination of HTML, CSS and Javascript in a meticulously planned, full application framework.
HTML5 is an API, a suite of tools to build applications.

We can thank mobile for HTML5

HTML5 looks set for rapid growth in 2012, driven by the need for cross-platform development as a result of increasing mobile market fragmentation.
